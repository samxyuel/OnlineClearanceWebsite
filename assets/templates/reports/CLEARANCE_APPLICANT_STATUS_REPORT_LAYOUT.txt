================================================================================
KEY DIFFERENCES FROM PROGRESS REPORT
================================================================================

REPORT TITLE:
- Changed from "Clearance Form Progress Report" to "Clearance Applicant Status Report"

SUMMARY SECTION:
- Changed title from "SUMMARY OF CLEARANCE FORMS" to "SUMMARY OF APPLICANT STATUS"
- Changed metrics from:
  * Total Forms (same)
  * Total Unapplied → Total Pending
  * Total In Progress → Total Approved
  * Total Completed → Total Rejected

TABLE SECTION - STUDENT:
- Removed: "Status" column (FormStatus: Unapplied/In Progress/Complete)
- Added: "Action Status" column (Pending/Approved/Rejected)
- Added: "Date Signed" column (YYYY-MM-DD format, or "Pending" text)
- Columns: Student No | Full Name | Department | Year Level | Section | Action Status | Date Signed

TABLE SECTION - FACULTY:
- Removed: "Status" column (FormStatus: Unapplied/In Progress/Complete)
- Added: "Action Status" column (Pending/Approved/Rejected)
- Added: "Date Signed" column (YYYY-MM-DD format, or "Pending" text)
- Columns: Employee No | Full Name | Employment Status | Department | Action Status | Date Signed

================================================================================
NEW PLACEHOLDERS FOR APPLICANT STATUS REPORT
================================================================================

${TotalPending} - Total number of applicants with action = 'Pending'
                  - Source: COUNT from clearance_signatories where action = 'Pending'

${TotalApproved} - Total number of applicants with action = 'Approved'
                   - Source: COUNT from clearance_signatories where action = 'Approved'

${TotalRejected} - Total number of applicants with action = 'Rejected'
                   - Source: COUNT from clearance_signatories where action = 'Rejected'

${ActionStatus} - Action status from clearance_signatories table
                  - Values: "Pending", "Approved", "Rejected"
                  - Source: clearance_signatories.action
                  - Display: As-is (no formatting needed)

${DateSigned} - Date when the action was taken
                - Format: YYYY-MM-DD (e.g., "2025-01-15")
                - Source: clearance_signatories.date_signed
                - Display: If NULL/empty → Show "Pending" text
                           If has value → Show date in YYYY-MM-DD format

================================================================================
TABLE COLUMN SPECIFICATIONS
================================================================================

STUDENT TABLE COLUMNS:
1. Student No: 25mm
2. Full Name: 45mm
3. Department: 35mm (7pt font, MultiCell for long names)
4. Year Level: 25mm
5. Section: 20mm
6. Action Status: 30mm (Pending/Approved/Rejected)
7. Date Signed: 30mm (YYYY-MM-DD or "Pending")

FACULTY TABLE COLUMNS:
1. Employee No: 25mm
2. Full Name: 50mm
3. Employment Status: 35mm
4. Department: 37mm (7pt font, MultiCell for long names)
5. Action Status: 30mm (Pending/Approved/Rejected)
6. Date Signed: 30mm (YYYY-MM-DD or "Pending")

Total table width: ~210mm (Student) / ~207mm (Faculty)

================================================================================
DATA SOURCE & FILTERING
================================================================================

PRIMARY DATA SOURCE:
- clearance_signatories table
- Filtered by:
  * user_id = logged-in signatory's user ID
  * designation_id = signatory's designation(s) for the selected period
  * Related to selected clearance period

JOIN PATH:
- clearance_signatories
  → clearance_forms (via clearance_form_id or similar)
  → students / faculty (via user_id or applicant_user_id)
  → users (for full name, etc.)

ACTION STATUS VALUES:
- "Pending" - No action taken yet (date_signed is NULL)
- "Approved" - Signatory approved the clearance
- "Rejected" - Signatory rejected the clearance

DATE SIGNED DISPLAY:
- If date_signed IS NULL or empty → Display "Pending" (text)
- If date_signed has value → Display as YYYY-MM-DD format

================================================================================
SUMMARY CALCULATIONS
================================================================================

Total Forms: COUNT(*) of all records in clearance_signatories 
             filtered by user_id, designation_id, and period

Total Pending: COUNT(*) WHERE action = 'Pending'

Total Approved: COUNT(*) WHERE action = 'Approved'

Total Rejected: COUNT(*) WHERE action = 'Rejected'

Note: Total Forms should equal Total Pending + Total Approved + Total Rejected
